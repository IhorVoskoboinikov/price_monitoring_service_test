Спроектировать приложение (как минимум - описать модели, эндпоинты, необходимые классы и функции, которые должны быть реализованы), которое будет реализовать api для сервиса по отслеживанию динамики цен на товары на различных сайтах.
Пользователь регистрируется на сайте по паре email+пароль (реализовать сам процесс регистрации и входа не нужно)
Пользователь каким-то способом, который в тестовом задании опускается, создаёт список товаров, цены на которые он будет отслеживать.
Сервис работает с API магазинов https://dummyjson.com/products и https://fakestoreapi.com/products, получая из них цены на товары.
Сервис должен получать текущие и исторические курсы валют (Украина), нужно выбрать api для их получения.
Предполагается, что в сервисе будут такие страницы, для которых нужно спроектировать API. Должна быть возможность указать валюту.
Страница получения списка товаров с полями:
название
диапазон цен на сегодня от/до
признак роста / падения / того же уровня цены по сравнению со средней ценой за предыдущие 30 дней
Сортировка списка: по цене, по тенденции цены (рост/падение).
Страница товара: название, описание товара, диапазон цен от и до. Дополнительно на странице товара есть кнопки/опции:
“Отобразить все цены”. Отображение списка пар “магазин-цена” на сегодня
“Отобразить историю цен”. Отображение графика изменения цен, на графике отображается динамика изменения цен в разных магазинах и средней цены
“Прислать уведомление на email, если цена опустилась ниже указанной цены”. В этом случае пользователь должен также указать цену товара для отправки email.

Вопросы, которые можно оставить “до уточнения”
цены указаны в долларах
цены и история цен должны отображаться в выбранной пользователем валюте
список магазинов может расширяться
история цен не ограничена по времени - считаем, что несколько лет может храниться
история цен в разных магазинах может быть разной (разные периоды)
товаров в дальнейшем в магазине может стать очень много (сотни тысяч, миллионы)
каждый товар может присутствовать только в части магазинов, или во всех. Для тестового задания можно совмещать id из разных API произвольным образом
Описание товара можно выбирать из любого из представленных API
